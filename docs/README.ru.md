# EVE Settings Manager

**EVE Online Settings Manager** — десктопное приложение для управления локальными файлами настроек EVE Online: копируйте раскладки интерфейса между персонажами, создавайте резервные копии и восстанавливайте профили, добавляйте заметки к аккаунтам — без запуска игрового клиента.

## Возможности

- **Переключение серверов** — Tranquility, Serenity, Singularity, Infinity, Thunderdome; автоопределение из папки установки EVE
- **Управление профилями** — создание, переименование, дублирование и удаление профилей настроек (подпапки `settings_*`, используемые EVE)
- **Список персонажей** — отображает все файлы персонажей с именами, разрешёнными через ESI API, и временем последнего изменения
- **Список аккаунтов** — отображает все файлы аккаунтов; добавляйте личные заметки к каждому аккаунту
- **Синхронизация настроек** — копируйте раскладку интерфейса одного персонажа или аккаунта сразу нескольким другим одним кликом
- **Резервная копия файла** — сохраните отдельный файл персонажа или аккаунта; восстановите одним кликом
- **Резервная копия папки** — создайте именованный снимок всего профиля; восстановите или удалите из боковой панели
- **Тёмная / светлая тема** — по умолчанию тёмная; при первом запуске следует системной теме; предпочтение сохраняется между сессиями

## Начало работы

1. Перейдите на страницу [Releases](https://github.com/mintnick/eve-settings-manager/releases).
2. Загрузите файл для вашей платформы:
   - **macOS:** `.dmg` — откройте и перетащите приложение в папку Applications
   - **Windows:** `.exe` — запустите напрямую, установка не требуется

> **Примечание для macOS:** Приложение ещё не подписано кодом. При первом запуске macOS может сообщить, что оно «повреждено и не может быть открыто». Наиболее надёжный способ исправить это — выполнить следующую команду в Терминале, а затем открыть приложение обычным способом:
> ```bash
> xattr -cr "/Applications/EVE Settings Manager.app"
> ```
> Также в течение примерно 1 часа после заблокированного запуска в разделе Системные настройки → Конфиденциальность и безопасность появится кнопка **Всё равно открыть**. Если кнопка не отображается, используйте команду Терминала выше.

## Хранение данных

Все постоянные данные хранятся локально — ничто не отправляется на внешние серверы.

| Платформа | Конфигурация | Резервные копии |
|---|---|---|
| macOS | `~/Library/Application Support/eve-settings-manager/config.json` | `~/Library/Application Support/eve-settings-manager/backups/` |
| Windows | `%APPDATA%\eve-settings-manager\config.json` | `%APPDATA%\eve-settings-manager\backups\` |

---

## Удаление

- **macOS:** Удалите приложение из папки Applications. Локальная папка с данными (содержащая конфигурацию и все резервные копии) **не удаляется автоматически** — при необходимости удалите её вручную: `~/Library/Application Support/eve-settings-manager`
- **Windows:** Удалите файл `.exe`. Для полного удаления также удалите папку с данными (содержащую конфигурацию и все резервные копии): `%APPDATA%\eve-settings-manager`

## Сборка из исходного кода

**Требования:** Node.js 18+, pnpm

```bash
git clone https://github.com/mintnick/eve-settings-manager.git
cd eve-settings-manager
pnpm install
pnpm dev        # запуск dev-сервера + Electron (горячая перезагрузка)
```

```bash
pnpm build      # проверка типов, сборка и упаковка через electron-builder
```

Результат записывается в `release/<версия>/`.
